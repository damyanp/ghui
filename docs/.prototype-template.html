Recipe grammar & available fields

A recipe is an ordered list of axes separated by →, ->, >, or ,. Each axis is one of:

AxisMeaning
Pivot(field) Bucket items by their own value of field. Items appear once per value (multi-valued items appear in multiple buckets).
Group(field) In this prototype, behaves like Pivot at the current scope. Logically distinct from Pivot in the design doc: Group is meant for sub-grouping inside an existing scope.
Hierarchy Render the parent ↔ sub-issue tree. Recursive — children of each node are organised using the rest of the recipe (the recipe wraps around at each tree level).
Sort(field) Sort items in the current scope by the field's natural order. No grouping.

Fields you can use: Epic, Workstream, Status, Iteration, Kind, Assignee, IssueType, State, Repository, Priority, Blocked, Estimate, Type (= issue / pullRequest / draftIssue).

Examples:

  • Pivot(Epic) → Hierarchy — today's behaviour.
  • Hierarchy → Group(Workstream) — root tree, sub-grouped by workstream.
  • Pivot(Status) → Pivot(Assignee) — kanban-ish board, two axes deep.
  • Sort(Priority) → Hierarchy — tree, but ordered by priority within each level.
